# ЗИОВПО. Лабораторная работа 1

## Задание 1. Подготовка репозитория

Репозиторий подготовлен для серверной части проекта: `operatorsha/ziovpo_labi`.

В проект перенесены и настроены:

- аутентификация через JWT access/refresh tokens;
- авторизация через ролевую модель `USER` / `ADMIN`;
- правила доступа к маршрутам;
- HTTPS через настройки Spring Boot;
- подключение к PostgreSQL через Spring Data JPA;
- переменные окружения для БД, JWT и HTTPS;
- pipeline GitHub Actions со шагами `test` и `build`.

## Переменные окружения

Пример находится в `.env.example`.

Основные переменные:

```env
SERVER_PORT=8443
DB_URL=jdbc:postgresql://localhost:5432/ziovpo_labi
DB_USERNAME=ziovpo
DB_PASSWORD=change-me
JWT_SECRET=change-me-to-a-random-64-character-secret-value-for-hs256
SSL_ENABLED=false
SSL_KEYSTORE_PATH=classpath:server.p12
SSL_KEYSTORE_PASSWORD=change-me
SSL_KEY_ALIAS=ziovpo
```

## Pipeline

Pipeline находится в `.github/workflows/ci.yml`.

Шаги:

- `test` - запуск тестов;
- `build` - сборка проекта после успешного прохождения тестов.

## UML-диаграммы

UML используется для описания структуры и поведения системы.

Основные виды UML-диаграмм:

- диаграмма классов - структура классов и связи между ними;
- диаграмма вариантов использования - пользователи и их сценарии;
- диаграмма последовательности - обмен сообщениями во времени;
- диаграмма активности - последовательность действий в процессе;
- диаграмма состояний - состояния объекта и переходы;
- диаграмма компонентов - модули системы и зависимости;
- диаграмма развертывания - размещение системы на узлах инфраструктуры.

## ER-диаграммы

ER-диаграмма описывает структуру базы данных: сущности, поля и связи.

Для текущей серверной части основная модель:

- `users` - пользователи;
- `user_sessions` - refresh-сессии пользователей;
- связь `users 1:N user_sessions`.
